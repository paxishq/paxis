# Paxis — Domain Language

> Shared vocabulary for the AI agent and the team.
> One term = one definition. Update whenever a new concept gets a name.

---

## Core Concepts

| Term | Definition |
|------|------------|
| Two-sided network | The core business model: enterprises (paying) and suppliers (free) connected in one compliance network |
| Enterprise node | A paying enterprise customer using Paxis to collect Scope 3 data and generate ESRS reports |
| Supplier node | A free supplier account onboarded by an enterprise to respond to questionnaires and maintain compliance docs |
| Compliance module | A pluggable regulation handler (CSRD, EU AI Act, CSDDD, CBAM…) that runs on the existing supplier network |
| Audit trail | The append-only Postgres `audit_log` table recording every agent action — the compliance product |
| LLM provider | The active large language model backend, switchable via `LLM_PROVIDER` (`gemini` or `featherless`) |
| MCP server | A second `Bun.serve()` process (port 15151 dev / unix socket prod) exposing 14 supplier compliance tools via the Model Context Protocol; authenticated by Bearer token |

## Domain Objects

| Term | Definition |
|------|------------|
| Questionnaire | A CSRD supplier data request dispatched by an enterprise to one or more suppliers |
| Questionnaire response | A supplier's answers to a dispatched questionnaire |
| AI inventory | A per-supplier record of AI tools classified by EU AI Act risk tier |
| Carbon entry | A Scope 1 or Scope 2 emissions record, typically parsed from an energy bill via Gemini Flash |
| Scope 3 aggregate | An enterprise-level aggregation of all supplier Scope 3 data for CSRD filing |
| ESRS report | An audit-ready CSRD-standard output document generated by the ESRS Report Agent |
| Supply chain relationship | The many-to-many link between an enterprise and its supplier nodes (`enterprise_suppliers` table) |
| MCP token | A supplier-issued Bearer token for external AI agent access to the MCP server; raw token shown once, SHA-256 hash stored only |
| Pending action | A proposed write operation emitted by the in-portal assistant as a `<PENDING_ACTION>JSON</PENDING_ACTION>` block; requires explicit supplier confirmation before execution |

## States

| Term | Definition |
|------|------------|
| Draft | A questionnaire or report in preparation, not yet dispatched |
| Dispatched | A questionnaire sent to supplier nodes, awaiting responses |
| Responded | A supplier has submitted answers to a questionnaire |
| Aggregated | Scope 3 data collected and rolled up into an enterprise aggregate |
| Audit-ready | An ESRS report that has passed all validation checks and is ready for regulatory filing |

## Abbreviations

| Abbrev | Expands to |
|--------|------------|
| CSRD | Corporate Sustainability Reporting Directive — EU law requiring large enterprises to report ESG data annually |
| ESRS | European Sustainability Reporting Standards — the reporting format mandated by CSRD |
| CSDDD | Corporate Sustainability Due Diligence Directive — human rights and environmental due diligence (2027) |
| CBAM | Carbon Border Adjustment Mechanism — carbon cost on imports; requires supplier emissions certificates |
| EU AI Act | EU regulation classifying AI systems by risk tier; requires documentation and audit trails |
| Scope 1 | Direct emissions from owned/controlled sources |
| Scope 2 | Indirect emissions from purchased energy |
| Scope 3 | All other indirect emissions across the value chain (requires supplier primary data) |
| CAC | Customer Acquisition Cost |
| ADR | Architectural Decision Record |
| IaC | Infrastructure as Code |
